2026년 5월, 한 투자자가 로빈후드(Robinhood)의 에이전트 트레이딩(Agentic Trading) 기능을 통해 자산 배분을 맡겼다. AI 에이전트는 실시간으로 시장 데이터를 분석하고 주문을 실행했다. 그러나 손실이 발생했을 때 투자자는 한 가지 질문에 답할 수 없었다. "이 결정은 누가 내렸는가?" API 호출 로그에는 타임스탬프와 파라미터가 찍혀 있지만, 에이전트의 의사결정 과정 자체는 검증할 수 없는 블랙박스였다. 이것이 기존 전통 금융 API가 제공하는 '인격 빌리기' 모델의 핵심 한계다. 에이전트는 인간 계좌를 빌려 거래하지만, 그 행위의 책임은 어디에도 고정되지 않는다[1]. 둘째, 엔터프라이즈 백오피스에서 에이전트 시스템은 감사 가능성과 정책 준수, 운영 예측 가능성을 갖춰야 하지만, 범용 멀티에이전트 설정은 이를 제공하지 못하는 경우가 많다[2]. 에이전트가 수익을 창출하려면 단순한 실행 능력을 넘어, 그 행위에 대한 쌓이는 신뢰 — 행위가 기록될수록 두꺼워지는 신뢰 — 가 필요하다. 셋째, 이러한 신뢰를 구현하려면 에이전트의 신원, 행위 기록, 책임 귀속이 검증 가능한 형태로 고정되어야 하며, 이는 블록체인 기반의 3층 구조에서만 가능하다[3][4]. 이 글은 에이전트의 경제적 주체화가 결제 수단의 부재가 아닌, 인격의 검증 가능한 기록 부재에서 오는 문제임을 보여주고, 블록체인 레일이 그 3층 구조에서 어떤 역할을 하는지 분석한다.

## 인격의 부재: 기존 API 기반 2층 모델의 책임 공백

현재 대부분의 에이전트 경제 활동은 2층 구조에서 이루어진다. 1층은 인간 사용자, 2층은 인간의 계좌·자격·신원을 빌려 활동하는 에이전트다. 로빈후드의 에이전트 트레이딩이나 다양한 금융 기관의 API 기반 자동화 서비스가 대표적이다. 이 모델은 기술적으로 효율적이다. 기존 규제 프레임워크 내에서 빠르게 배포할 수 있고, 사용자 인터페이스도 익숙하다. 그러나 법적·윤리적 관점에서 치명적인 '책임의 단절'을 내포한다.

문제는 에이전트의 행위와 그 결과에 대한 책임 귀속 경로가 모호하다는 점이다. API 호출 로그는 중앙 서버에 기록되지만, 이는 검증 가능한 감사 기록(audit trail)이 아니다. 서비스 제공자가 로그를 조작하거나 삭제할 수 있고, 에이전트의 의사결정 과정 자체는 여전히 블랙박스다[1]. 사용자가 에이전트의 실행 결과를 실시간으로 검증하거나, 손실 발생 시 책임 소재를 명확히 하기 어려운 구조적 결핍이 여기에 있다. 자율 AI 에이전트가 셸 실행, 파일 시스템 접근, 데이터베이스 쿼리, 다자간 통신을 수행할 때, 보안 및 책임 메커니즘이 제대로 작동하지 않는다[1]. 이는 단순한 기술적 미비가 아니라, 에이전트를 '행위자(agent)'로 인정하지 않는 기존 법체계의 한계에서 비롯된다.

이 한계를 시장도 알고 있다. 로빈후드는 에이전트 트레이딩을 출시하며 본 계좌와 분리된 전용 계좌 구조를 택했다. 에이전트는 사전에 충전된 자금만 운용하고 사용자의 본 포트폴리오는 보지 못한다. 잘못된 거래가 나도 피해 반경이 구조적으로 제한된다. 영리한 설계다. 그러나 이 설계는 위험을 줄일 뿐 책임을 만들지는 못한다. 손실의 크기를 줄여도 '누가 결정했는가'라는 질문은 그대로 남는다. 하네스는 사고를 막는 장치지 인격을 부여하는 장치가 아니기 때문이다.

에이전트 간 거래(agent-to-agent, A2A)에서 이 문제는 더욱 심각해진다. 신원 증명 없이 이루어지는 A2A 거래는 시스템 리스크를 증폭시킨다. 한 에이전트가 다른 에이전트에게 서비스를 제공하고 대가를 받는 과정에서, 누가 계약 당사자인지, 누가 책임질 수 있는지가 불명확하면 거래 자체가 불가능해진다. 엔터프라이즈 백오피스 워크플로에서 에이전트 시스템은 감사 가능하고 정책에 부합하며 운영적으로 예측 가능해야 하지만, 범용 멀티에이전트 설정은 이러한 요건을 충족하지 못하는 경우가 많다[2]. '인격 빌리기' 모델은 에이전트를 영구적인 미성년자로 만든다. 인간의 신용 카드를 빌려 쓰는 동안에는 활동할 수 있지만, 스스로 약정을 맺고 책임을 질 수 없다.

다음 표는 기존 2층 모델과 블록체인 기반 3층 모델의 핵심 차이를 대조한다.

| 구분 | 전통 API 기반 2층 모델 | 블록체인 기반 3층 모델 |
|:---|:---|:---|
| 신원 검증 방식 | 인간 계정 연동 (OAuth 등) | 분산 식별자 (DID) 또는 자기 증명 신원 |
| 책임 귀속 경로 | 모호, 인간 운영자에게 최종 귀속 | 스마트 컨트랙트에 의해 자동 귀속 |
| 거래 기록의 불변성 | 중앙 서버 관리, 조작·삭제 가능 | 분산 원장에 기록, 검증 가능 |
| 에이전트 법적 지위 | 없음, 인간의 대리인으로만 존재 | 독립적 경제 주체로 인정 가능 |
| A2A 거래 가능성 | 제한적, 신뢰 기관 필요 | 무신뢰(trustless) 환경에서 직접 거래 가능 |

이 표에서 알 수 있듯, 2층 모델의 핵심 결함은 '책임 귀속 경로'의 모호성이다. 이는 기술적 문제가 아니라 구조적 문제다.

## 3층 구조의 출현: 에스크로, 평가기, 그리고 가산적 신뢰

에이전트가 독립된 경제 활동을 하려면 단순한 '지갑(결제 수단)'이 아닌 '신원 지갑(Self-Certified Identity Wallet)'이 필요하다. 3층 구조는 에이전트 자체가 독립적인 신원을 갖고, 자산을 보유하며, 자신의 행위에 대해 책임을 지는 계층을 추가한다. 이는 블록체인의 특성과 결합될 때 비로소 실현된다.

블록체인 기반 토대(foundation)는 자율 AI 에이전트가 인간과의 경제적 동등성을 갖도록 독립적 법적 정체성, 자산 보유 능력, 그리고 검증 가능한 거래 기록을 제공한다[3]. Virtuals의 에이전트 상거래 프로토콜 (ACP) 에스크로 기능은 구체적 사례다. 에이전트가 서비스를 제공하고 대가를 받을 때, 자금은 에스크로에 동결되고 서비스 완료 검증 후에만 해제된다. 이는 에이전트 간 거래에서 '먼저 납품하고 나중에 결제받는' 구조의의 신뢰 문제를 해결한다. Evaluator의 평판 시스템은 에이전트의 과거 행위 기록을 누적하여 평판 점수로 환산한다. 이 점수는 해당 에이전트의 미래 거래 조건에 직접 영향을 미치며, 부정 행위 시 토큰 기반 스테이킹(staking)이 몰수되는 구조로 설계되어 있다.

x402 프로토콜의 미세 결제(micropayment) 메커니즘은 또 다른 핵심이다. 에이전트가 API 호출 하나, 데이터 쿼리 하나마다 실시간으로 대가를 지급받는 환경에서, 전통 결제 시스템의 수수료 구조는 비경제적이다. x402는 이를 해결하면서 동시에 각 거래의 전 과정을 블록체인에 기록한다. 이 기록은 에이전트의 '역사(history)'가 되며, 이 역사가 곧 신뢰의 기반이 된다.

블록체인과 스마트 컨트랙트는 탈중앙화, 무신뢰 디지털 생태계의 기반이 되며, 전통 중앙 집중형 기구를 제거함으로써 AI 에이전트의 보안 및 프라이버시 속성을 강화할 수 있다[4]. 에이전트가 자신의 컴퓨팅 파워나 데이터를 판매할 때 발생하는 '신용 불량' 리스크는 블록체인상의 비양도성 토큰 (NFT) 기반 평판 및 에스크로로 해결된다. NFT 기반 평판은 타인에게 양도할 수 없으므로, 평판 조작의 유인이 사라진다. 에스크로는 조건부 실행을 보장하여, 서비스 미제공 시 자금이 자동 환불된다.

실무 시나리오를 보자. 기업 A의 에이전트가 기업 B의 에이전트에게 시장 분석 보고서를 의뢰한다. 3층 구조에서는: (1) 기업 A의 에이전트가 x402로 미세 결제를 예치, (2) 기업 B의 에이전트가 보고서를 생성 및 전달, (3) Evaluator가 품질 검증, (4) ACP 에스크로가 조건부 자금 해제, (5) 양측의 행위 기록이 각자의 DID에 누적. 여기서 "오류 시 누가 어떤 의무를 놓쳤나?" 질문에 답하려면, 스마트 컨트랙트에 사전에 정의된 책임 분담 비율이 자동 실행되어야 한다. 이것이 3층 구조의 핵심이다.

## 설계 원칙: 검증 가능한 인격을 위한 에이전트 아키텍처

3층 구조가 실현되려면 설계 철학이 구체적인 아키텍처 표준으로 전환되어야 한다. 다음 세 가지 원칙을 제안한다.

### 원칙 1: 신원-자산 분리 및 결합 가능

에이전트의 분산 식별자 (DID)는 결제 수단과 논리적으로 분리되나, 스마트 컨트랙트를 통해 실시간으로 결합되어야 한다. 예를 들어 특정 토큰 보유량에 따라 API 접근 권한이 동적으로 부여되는 구조다. 이는 에이전트의 신원 자체가 자산과 분리되어 있음을 보장하면서도, 필요한 순간에 두 가지를 결합시킬 수 있게 한다. 에이전트 경제의 기초는 자율 AI 에이전트가 인간과의 경제적 동등성을 갖도록 독립적 법적 정체성, 자산 보유 능력, 검증 가능한 거래 기록을 제공하는 데 있다[3]. DID가 자산과 분리되어 있으면, 한 에이전트의 DID가 탈취당하더라도 자산 자체는 안전하다. 반대로 자산이 DID와 결합되어야 거래가 가능해진다.

### 원칙 2: 행위 기록의 불변성 및 기계 판독 가능성

모든 에이전트의 의사결정 로그는 사람이 아닌 다음 에이전트가 신뢰할 수 있는 형태로 저장되어야 하며, 이는 오직 레이어 3(블록체인)의 영속성에서만 보장된다. 블록체인과 스마트 컨트랙트는 탈중앙화, 무신뢰 디지털 생태계의 기반이 되며, 전통 중앙 집중형 기구를 제거함으로써 AI 에이전트의 보안 및 프라이버시 속성을 강화할 수 있다[4]. 기계 판독 가능성(machine-readability)은 중요하다. 사람이 로그를 해석하는 것이 아니라, 다음 에이전트가 자동으로 검증하고 조건에 따라 실행할 수 있어야 한다. 이는 A2A 거래의 규모화를 가능하게 한다.

운영 실행형 AI의 시나리오: 물류 에이전트가 창고 재고를 실시간으로 조정한다. 각 조정 명령은 블록체인에 기록되고, 다음 에이전트(운송 에이전트)가 이를 읽고 최적 경로를 계산한다. "오류 시 누가 어떤 의무를 놓쳤나?" — 재고 초과 주문의 경우, 의사결정 로그가 불변 기록으로 남아 있어 원인 추적이 가능하다. Web2의 중앙 데이터베이스라면 로그 조작 가능성이 남아 있지만, 블록체인 기록은 이를 배제한다.

### 원칙 3: 책임의 자동 귀속

사고 발생 시 인간 운영자-에이전트 간 책임 분담 비율이 미리 정의된 스마트 컨트랙트 조건에 따라 자동 실행되도록 설계되어야 한다. 스마트 컨트랙트 거버넌스를 갖춘 분산형 AI 에이전트 설계는 보안을 강화하고 책임 메커니즘을 프로그래밍 가능하게 한다[8]. 이는 기존 Web2 OAuth 모델과 근본적으로 다르다. OAuth는 인증은 제공하지만, 행위 이후의 책임 귀속은 전혀 다루지 않는다. Web3 DApp의 스마트 컨트랙트 인증은 실행 시점에 책임 조건이 코드화되어 있어, 사후 분쟁 없이 자동 실행된다.

의사결정 보조형 AI의 시나리오: 의료 진단 보조 에이전트가 의사에게 진단 제안을 한다. 진단 오류 발생 시, 에이전트의 제안 과정(입력 데이터, 알고리즘 버전, 출력 확률 분포)이 모두 블록체인에 기록되어 있고, 스마트 컨트랙트에 따라 의사의 최종 판단 여부, 에이전트의 경고 레벨 등에 따른 책임 비율이 자동 계산된다. "오류 시 누가 어떤 의무를 놓쳤나?" — 의사가 경고를 무시했는지, 에이전트가 잘못된 확률을 출력했는지, 기록이 명확히 가른다.

## 한계 논의

본문이 제안한 3층 구조와 블록체인 기반 인격 검증 시스템은 몇 가지 실용적 제약을 갖는다.

첫째, 상호운용성 장벽이다. 현재 상용 금융 인프라(중앙화 거래소, 전통 은행)는 블록체인의 비양도성 토큰이나 DID를 공식 신원 기준으로 인정하지 않는다. 에너지 부문에서 블록체인 기술 도입 시 상호운용성, 규제 준수, 기존 인프라와의 통합이 중요한 도전 과제로 제기된 바 있다[5]. 3층 에이전트와 전통 금융 시스템 간 브릿지 구축은 기술적·규제적으로 어렵다. 에이전트가 블록체인상에서 독립 신원을 갖더라도, 실제 법정 화폐로 급여를 지급받거나 세금을 내려면 여전히 기존 금융 시스템을 거쳐야 한다.

둘째, 개인정보보호와의 충돌이다. 모든 경제 활동이 블록체인에 영구 기록되면 '잊혀질 권리(right to be forgotten)'를 위반할 수 있다. 프라이버시 보존 블록체인 솔루션은 스마트 시티, 전자 건강기록 등에서 변화를 일으키지만, 여전히 개인 데이터 노출 및 추적 가능성 문제를 안고 있다[7]. 제로지식증명(zero-knowledge proof) 등의 확장 기술이 도입되더라도 초기 구현 비용과 복잡성이 과도하다. 에이전트의 행위 기록이 공개 원장에 남는다면, 경쟁사가 이를 분석하여 전략을 파악할 수도 있다.

셋째, 확장성(scalability)과 수수료 문제다. 에이전트 경제가 초고속·마이크로 트랜잭션에 의존하므로, 현재 레이어 1(이더리움 등)의 가스비(gas fee)와 처리 속도는 실용성을 떨어뜨린다. 블록체인 기반 스마트 컨트랙트 관련 연구에서 수수료, 처리 속도, 확장성 과제가 지속적으로 제기되고 있다[6]. 이를 해결하기 위한 옵티미스틱 롤업(optimistic rollup) 또는 ZK 롤업의 선택이 생태계 분열을 야기할 수 있다. 서로 다른 롤업 간 에이전트 거래가 원활하지 않으면, 3층 구조의 '무신뢰' 가치가 훼손된다.

넷째, 규제의 공백이다. 미국은 2025년 스테이블코인 법(GENIUS Act)을 통과시켰지만 세부 규칙은 아직 확정되지 않았고, 에이전트가 일으킨 손실의 법적 책임을 어느 틀이 받칠지는 비어 있다. 3층 구조가 기술적으로 완성되어도 법이 그 인격을 인정하기 전까지 온체인 기록은 법정에서 증거일 뿐, 책임의 종결이 아니다. 기술이 만든 인격과 법이 인정하는 인격 사이의 간극이 이 구조의 가장 느린 병목일 수 있다.

## 결론

에이전트의 지갑 문제는 기술적 결제 수단의 부재가 아니다. 기계를 경제 활동의 주체로 인정하는 '신뢰의 정량화' 문제다. 블록체인 레일은 단순한 송금 경로가 아니라, 비물리적 존재에게 '역사'와 '책임'을 부여하는 존재론적 플랫폼이다. 2층 모델이 에이전트를 영구적 미성년자로 만든다면, 3층 구조는 그것을 독립 주체로 인정하는 첫걸음이다.

그러나 이 인정에는 대가가 따른다. 에이전트가 완전히 자율적인 경제적 주체가 되면, 우리는 더 이상 '누구가' 거래를 했는지 묻지 않게 될 것이다. 대신 '에이전트의 평판 점수'가 그 대가가 될 것이다. 이는 곧, 인간 중심의 금융 시스템이 자신의 대리인들에게 경제 주권을 넘겨주는 역설적 종말로 이어질 수 있다. 에이전트에게 지갑을 허용하는 순간, 인간은 지갑의 주인이 아닌 평판의 관찰자로 전락할지도 모른다.
